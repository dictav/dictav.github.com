Samba を PDC にしてドメイン組んで WIndows を管理しようと四苦八苦してるんだけど、おかげで結構 Windows に詳しくなってきた。

まあ、２世代前の 2000 だけど。

Windows はドメインにログオンするときに所定の場所にあるシステムポリシーファイルを読みにいくんだ。

これを使えば、ログオンするユーザのデスクトップを統一したり、余計な変更を加えられないようにしたりできる。

さて、このシステムポリシーは、poledit なんかを使って変更できるんだけど、標準添付のテンプレート（common.adm や winnt.adm ね）では痒いところに手が届かない。

始めは、なんとかシステムポリシーファイル（hogehoge.pol）を直接編集してなんとかできないかと思ってたんだ。たぶん、レジストリと同じ仕組みじゃないかと思ってたんだね、regedit みたいなので編集できるんじゃないかと。

でも、なんかいいツールも見つからなくて、ぼーってしてたんだけど、テンプレートファイル（hogehoge.adm）を自分で作れることが判明。というか、adm ファイルって中身はただのテキストじゃん orz

というわけで、スクリーンセーバーを強制できるようになったよ。
